💡 The Logic Behind the Solution

Why start from the end?

The standard merge approach starts from the beginning. However, nums1 contains valid elements at the start and empty space (zeros) at the end. If we started merging at index 0, we would overwrite elements in nums1 that haven't been processed yet!


The "Reverse" Strategy:

  • We place pointers p1 and p2 at the end of the valid data.
  • We place pointer p at the very end of the nums1 array (the destination).
  • We compare the largest elements first. The winner goes to nums1[p].

Why is this optimal?

  • No Overwriting: By filling from the back, we utilize the empty space first. We only overwrite a position in nums1 after its original value has already been moved.
  • Space Complexity: O(1). We modify the array in-place without needing a temporary array to hold values.
  • Time Complexity: O(m + n). We process every element exactly once.
